11-1. 리턴값은 없고 인자는 있는 함수의 예(인자는 있음. 인자는 입력값을 의미)

11-2. 리턴값은 있고 인자는 없는 함수의 예
메인 메소드에서 A라는 메소드를 출력한다. 
A라는 메소드에서는 변수 B를 선언후 초기화한다. 그리고 그 B를 리턴한다. 그럼 B의 변수값이 출력된다.
즉 함수A를 불러오면 변수값B(리턴값)로 취급한다. 

11-3. 리턴값과 인자가 둘다 있는 함수의 예
A라는 메소드에서 int a와 int b라는 매개 변수를 가진다.
그리고 a + b = B 라는 연산을 하고 B는 리턴값이다.
메인 메소드에서는 A메소드를 불러와서 매개변수 a, b에 원하는 숫자를 입력하고 리턴값 B를 출력한다.(아니면 그냥 A함수 자체를 출력?) 

리턴값은 꼭 변수만 되는게 아니라 배열도 가능하다.

전역변수란

23. 상속(extends) 이란?

클래스 하나당 하나의 클래스를 상속 가능. 단, 상속의 상속은 가능(클래스>자식 클래스> 손자 클래스...손자 클래스라는 말은 없다..)

상속 방법
class 하위 클래스명 extends 상위 클래스명

26. 생성자 란?
객체가 생성되기 직전에 실행되는 메소드. 반환값이 없는 메소드 이다.

역할은 클래스의 필드변수들을 초기화 해준다.
즉, 필드변수들의 초기값을 입력한 함수다.
(필드 변수란 클래스 안에 있는데 메소드나 생성자 밖에 있는 변수, 즉 클래스를 제외한 뭔가에 속해있지 않은 변수다.)

예를 들면 
클래스가 RPG게임 캐릭터의 스테이터스를 정의 한다고 치면 공/방/마법/회피/운 이런식으로 클래스가 짜지는데 
이것들이 필드변수고 
클래스 내부에 생성자 라는 함수를 만들어서 변수값을 모두 5로 입력하면 5/5/5/5/5 이런식으로 필드변수들이 초기화가 된다. 이걸 기본값이라고 구상하면 외부에서 객체 생성후 따로 값을 입력하지 않으면 저렇게 기본 스펙으로 게임을 시작하는 거다.

클래스에 생성자를 따로 추가하지 않아도 Default 생성자가 자동으로 생성된다.
